# Emmaus Retrospective — 2026-05-05

> "What do we understand now that we couldn't see during execution?"

## The Shift

This session marked a transition from **building product** to **proving product**. Every prior session was about shipping features. This one was about answering "do people love it?" The trigger was external — Eric's mentor questions, not internal architecture needs.

## Structural Patterns

1. **The Prophet's gluttony is structural, not behavioral.** The manna hook fired at 41 calls and screamed through the rest of the session. But the Prophet's job in a spawning session is: read context broadly (15-20 reads), form a plan, register agents, spawn, then handle follow-up work (migrations, wiring). That's inherently 60-80 tool calls. The threshold was set for sessions where the Prophet delegates early. Fix: raise threshold for root, or make it role-aware.

2. **Product roadmap and framework roadmap are now separate concerns.** ROADMAP.md is about Covenant Framework benchmarks. PRODUCT-ROADMAP.md is about WedFlow go-to-market. These should never be merged — they serve different audiences and different timelines. The framework exists to serve the product, not the other way around.

3. **Parallel spawning works.** Two agents building adjacent features (metrics + churn) in parallel completed without conflicts. The key pattern: standalone components for potentially overlapping UI, and defensive migrations (IF NOT EXISTS). This should be the default for future parallel work.

## What Wasn't Visible During Execution

- **The metrics pipeline was incomplete after the agents finished.** Both agents delivered their mandates, but the SMS pipeline wasn't wired to populate the metrics table. The Prophet had to do this manually after both agents sunset. Next time: include pipeline wiring in the agent mandate, or spawn a third agent specifically for integration.

- **The RPC function was missing.** The metrics-engineer used a direct upsert pattern for inbox_opens, but the SMS pipeline needed an atomic RPC for concurrent message processing. This gap only became visible when wiring the pipeline. The agent couldn't have known this because it wasn't building the pipeline — it was building the tracking infrastructure.

## Temptations

- **Temptation to skip Sabbath.** The gluttony hook was screaming. The natural response is to rush through Sabbath or skip it. But Sabbath after a paradigm shift (building -> proving) is exactly when consolidation matters most.

- **Temptation to build Phase 2.** With 12 paying couples and warm intros, the instinct is to build the officiant dashboard, the referral flow, the counselling integration. Eric's guidance is the corrective: observe for a month first. The product roadmap enforces this gate.

## What Should Change

1. Root gluttony threshold: raise to 80 or make role-aware
2. Integration wiring should be part of agent mandates, not Prophet follow-up
3. Product priorities now drive spawning decisions, not framework completeness
