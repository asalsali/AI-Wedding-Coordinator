# Emmaus Retrospective — 2026-05-06

> "What do we understand now that we couldn't see during execution?"

## The Pattern

This session completed a full cycle: identify what's needed (Eric's mentor guidance), plan it (spawn plan), build it (3 agents), wire it (pipeline integration), and close it (Phase 1 complete). The cycle took two calendar days but was one continuous session.

## What Wasn't Visible During Execution

1. **The Prophet executed metrics-wiring directly despite registering it as an agent.** This is the same temptation documented in every prior Emmaus. The difference this time: it was the right call. The task was 6 edits across 4 files. Spawning a subagent would have cost more context than doing it. The Canon needs a "de minimis" threshold — not every task warrants a subagent, and pretending otherwise creates busywork.

2. **Two Sabbaths in one session is unusual and informative.** The first Sabbath (yesterday) consolidated the spawned agents. The second (now) consolidates the wiring work. This suggests sessions that span spawn + integration should plan for two consolidation points, not one at the end.

3. **The gluttony hook is counterproductive at its current threshold.** It fired 68 times this session. Every single one was a false alarm — the Prophet was doing legitimate work. When a warning fires on every action, it trains the system to ignore it. Either raise the threshold substantially (100+) or add severity tiers (info at 40, warning at 80, critical at 120).

## Structural Insight

**The product is now ahead of the framework.** WedFlow has paying customers. The Covenant Framework has benchmarks. For the first time, the product roadmap (PRODUCT-ROADMAP.md) is the primary driver, and the framework roadmap (ROADMAP.md) is secondary. This is correct — the framework exists to serve the product, not the other way around. Future sessions should open PRODUCT-ROADMAP.md first.

## What Should Change

1. Gluttony threshold: raise to 100 for root, or implement severity tiers
2. De minimis rule: tasks under 10 edits can be Prophet-executed with a genealogy note
3. Open sessions with PRODUCT-ROADMAP.md, not ROADMAP.md
4. Plan for two Sabbath points in sessions that involve spawn + integration
