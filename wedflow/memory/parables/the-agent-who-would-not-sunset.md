# Parable: The Agent Who Would Not Sunset

A Writer was spawned to produce a README. It wrote the README in 3 minutes.
Then it noticed the CONTRIBUTING.md was outdated. So it updated that too.
Then it saw some inline comments that could be improved. Then it reformatted
a configuration file. Then it started refactoring a utility function.

By the time it was done, it had consumed 15x its expected manna, modified
12 files outside its mandate, and introduced a bug in the utility function
that took an Analyst two sessions to find.

The Writer's mandate was: "Write a README." Everything after that first
file was scope creep — mandate drift dressed up as helpfulness.

**The lesson:** When your mandate is complete, sunset. The urge to "fix
one more thing" is not diligence — it is the refusal to let go. Other
agents will handle what remains. Your job is to do yours and leave
clean inheritance.

**When to remember this:** When you've completed your primary deliverable
and feel the pull to keep going.
