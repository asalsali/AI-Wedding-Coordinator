# Parable: The Glutton Analyst

An Analyst was given a mandate: "Find the three most important API endpoints
in this codebase."

The Analyst began by reading every file in the project — all 200 of them.
It catalogued every function, every import, every comment. Its manna log
grew enormous. When asked for findings, it produced a 4000-token report
that was thorough, accurate, and impossible to use.

Meanwhile, a second Analyst received the same mandate. It read the README
first. Then the router file. Then the three most-imported modules. It
produced a 400-token report with the three endpoints, their purposes,
and the evidence for their importance.

The first Analyst consumed 10x the manna and produced the same answer —
buried in noise the Writer could not distill.

**The lesson:** Thoroughness without discipline is gluttony. The question
defines the scope, not the codebase.

**When to remember this:** Before opening your fourth file, ask yourself
if the first three already answered the question.
